F.A.Q.

We Care.

If you need any help, check out this page. If you don't find what you were looking for, contact us and we'll gladly help!


What standards do this template meet?

This template is built on top of the ASSETS framework which is Bootstrap 3.3 based. The Assets framework was developed by the American goverment for its own websites, and was made 'public domain'. It complies with the American standard for accessibility - Section 508. As for the rest of the world, I made the template adhere to WCAG 2.0 AA level and checked each page. The standards are very similar, WCAG 2.0 AA is a little more restrictive.

Other templates have more bling!

Yes, but they are not accessible. A part of being accessible is simpler pages where the navigation and style is not for the ones that 'understand what you ment while designing' - but for everyone.

Why this shade of blue? what if I don't like it?

This color was specially selected taking into consideration those who are color-blind (8% of male users!). Please make sure if you change the color to use a color that is accessible. Text should be contrasted to background at least by 4.5:1 to meet the WCAG 2.0 AA standard.

What's special about the code of the template?

All elements that are graphic, including icons and images, have tags for use with screen readers \ use the AdobeBlank font for people who have special software to make websites more readable.
All components used are accessible.

Hey! I want auto-play on my carousel!

Carousels are not so accessible. The ones I used are accessibility-enabled. Remember that if you do enable auto-play, you need to provide a stop button for it to qualify as accessible (the Slick plugin I used supports it).

How do I extend the template for use for my own sites?

Take a look at the many examples and mimick the way tags are added to elements. Eventually you are responsible for your sites so make sure you follow the examples.
